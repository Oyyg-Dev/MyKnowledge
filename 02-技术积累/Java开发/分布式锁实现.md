# 分布式锁实现方案

## 概述
在分布式系统中，多个服务实例可能同时处理相同的请求，需要使用分布式锁来保证操作的原子性。

## 常见实现方式

### 1. 基于 Redis 的分布式锁（推荐）

#### 使用 Redisson 实现

**依赖配置：**
```xml
<dependency>
    <groupId>org.redisson</groupId>
    <artifactId>redisson-spring-boot-starter</artifactId>
    <version>3.17.0</version>
</dependency>
```

**配置类：**
```java
@Configuration
public class RedissonConfig {
    
    @Bean
    public RedissonClient redissonClient() {
        Config config = new Config();
        config.useSingleServer()
              .setAddress("redis://192.168.1.100:6379")
              .setPassword("your_password")
              .setDatabase(0);
        return Redisson.create(config);
    }
}
```

**使用示例：**
```java
@Service
public class OrderService {
    
    @Autowired
    private RedissonClient redissonClient;
    
    public void processOrder(String orderId) {
        String lockKey = "order:lock:" + orderId;
        RLock lock = redissonClient.getLock(lockKey);
        
        try {
            // 尝试获取锁，最多等待10秒，锁30秒后自动释放
            if (lock.tryLock(10, 30, TimeUnit.SECONDS)) {
                // 执行业务逻辑
                doProcessOrder(orderId);
            } else {
                throw new BusinessException("系统繁忙，请稍后重试");
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new BusinessException("处理中断");
        } finally {
            // 释放锁（仅当前线程持有时）
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
    
    private void doProcessOrder(String orderId) {
        // 业务逻辑
    }
}
```

#### 关键参数说明

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| waitTime | 等待获取锁的最大时间 | 10秒 |
| leaseTime | 锁自动释放时间 | 30秒（根据业务调整） |
| retryInterval | 重试间隔 | 默认即可 |

### 2. 基于数据库的分布式锁

**适用场景：** 没有 Redis 的情况下

**实现方式：**
```sql
-- 创建锁表
CREATE TABLE distributed_lock (
    lock_key VARCHAR(100) PRIMARY KEY,
    lock_value VARCHAR(100) NOT NULL,
    expire_time BIGINT NOT NULL,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

-- 创建索引
CREATE INDEX idx_expire_time ON distributed_lock(expire_time);
```

**Java 实现：**
```java
@Service
public class DbLockService {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;
    
    public boolean tryLock(String lockKey, String lockValue, long expireSeconds) {
        try {
            String sql = "INSERT INTO distributed_lock (lock_key, lock_value, expire_time) " +
                        "VALUES (?, ?, ?)";
            long expireTime = System.currentTimeMillis() + expireSeconds * 1000;
            jdbcTemplate.update(sql, lockKey, lockValue, expireTime);
            return true;
        } catch (DuplicateKeyException e) {
            // 锁已存在，检查是否过期
            return tryAcquireExpiredLock(lockKey, lockValue, expireSeconds);
        }
    }
    
    public void unlock(String lockKey, String lockValue) {
        String sql = "DELETE FROM distributed_lock WHERE lock_key = ? AND lock_value = ?";
        jdbcTemplate.update(sql, lockKey, lockValue);
    }
    
    private boolean tryAcquireExpiredLock(String lockKey, String lockValue, long expireSeconds) {
        String sql = "UPDATE distributed_lock SET lock_value = ?, expire_time = ? " +
                    "WHERE lock_key = ? AND expire_time < ?";
        long now = System.currentTimeMillis();
        long expireTime = now + expireSeconds * 1000;
        int updated = jdbcTemplate.update(sql, lockValue, expireTime, lockKey, now);
        return updated > 0;
    }
}
```

### 3. 基于 Zookeeper 的分布式锁

**适用场景：** 已有 Zookeeper 集群的系统

**优点：**
- 强一致性
- 自动释放（临时节点）
- 支持公平锁

**缺点：**
- 性能相对较低
- 依赖 Zookeeper 集群

## 最佳实践

### ✅ 正确做法

1. **设置合理的超时时间**
```java
// 根据业务处理时间设置 leaseTime
lock.tryLock(10, 30, TimeUnit.SECONDS);
```

2. **使用 finally 释放锁**
```java
try {
    if (lock.tryLock()) {
        // 业务逻辑
    }
} finally {
    if (lock.isHeldByCurrentThread()) {
        lock.unlock();
    }
}
```

3. **锁的粒度要细**
```java
// ✅ 好的做法：锁具体的订单
String lockKey = "order:lock:" + orderId;

// ❌ 不好的做法：锁整个订单模块
String lockKey = "order:lock";
```

4. **处理获取锁失败的情况**
```java
if (!lock.tryLock(10, 30, TimeUnit.SECONDS)) {
    // 明确告知用户
    throw new BusinessException("系统繁忙，请稍后重试");
}
```

### ❌ 常见错误

1. **忘记释放锁**
```java
// ❌ 错误：异常时锁不会释放
lock.lock();
doSomething();
lock.unlock();

// ✅ 正确：使用 try-finally
try {
    lock.lock();
    doSomething();
} finally {
    lock.unlock();
}
```

2. **锁超时时间设置不当**
```java
// ❌ 错误：业务需要 60 秒，但锁 30 秒就释放了
lock.tryLock(10, 30, TimeUnit.SECONDS);
longTimeOperation(); // 需要 60 秒
```

3. **不检查是否获取到锁**
```java
// ❌ 错误：没有检查返回值
lock.tryLock();
doSomething(); // 可能没有获取到锁就执行了

// ✅ 正确：检查返回值
if (lock.tryLock()) {
    doSomething();
}
```
