## 🚀 快速开始

### 创建新文档

#### 方法 1：使用模板
1. 右键点击目标文件夹
2. 选择 "新建笔记"
3. 复制对应模板内容

#### 方法 2：快捷键
- `Ctrl + N` - 新建笔记
- `Ctrl + O` - 快速切换笔记
- `Ctrl + P` - 命令面板

### 使用双向链接

在文档中使用 `[[文档名]]` 创建链接：

```markdown
参考 [[MS项目-后端开发]] 的配置方式
这个问题在 [[问题-2026-05-04-OA回调]] 中遇到过
```

### 使用标签

使用 `#标签名` 组织文档：

```markdown
#项目文档 #MS项目 #后端开发
#问题排查 #生产问题 #已解决
#技术积累 #Java #Spring
```

## 💡 使用技巧

### 1. 项目开发流程

**开发新功能时：**
1. 在项目文档中记录需求和设计
2. 开发过程中记录关键代码位置
3. 遇到问题使用问题排查模板
4. 完成后更新 API 文档

**示例：**
```markdown
# MS项目-生产计件功能

## 需求
[需求描述]

## 设计方案
[设计思路]

## 关键代码
- 后端: `ums-faret/xxx/XxxController.java:123`
- 前端: `faretms-ui/src/views/xxx.vue:45`

## API 变更
- [[API-生产计件查询]]
- [[API-生产计件提交]]

## 相关问题
- [[问题-2026-05-04-计件数据不准确]]
```

### 2. 问题排查流程

**遇到问题时：**
1. 使用问题排查模板创建文档
2. 记录问题现象和环境信息
3. 记录排查过程和日志
4. 解决后标记状态为 🟢已解决
5. 提炼经验到技术积累

### 3. 知识积累流程

**学习新技术时：**
1. 在技术积累中创建主题笔记
2. 记录核心概念和代码示例
3. 链接到实际项目应用
4. 定期回顾和更新

## 🔧 推荐插件配置

### Templater 配置
1. 设置 → Templater
2. Template folder location: `99-模板`
3. 启用 "Trigger Templater on new file creation"

### Calendar 配置
1. 设置 → Calendar
2. 设置日记存储位置: `04-日常笔记/2026`

### Obsidian Git 配置
1. 设置 → Obsidian Git
2. 设置自动提交间隔: 30 分钟
3. 启用 "Auto pull on startup"

## 📝 模板使用说明

### 项目文档模板
用于创建项目的基础文档，包含项目概述、目录结构、常用命令等。

**使用场景：** 新项目启动、项目交接、项目文档整理

### 问题排查模板
用于记录生产问题的排查过程和解决方案。

**使用场景：** 生产问题、Bug 修复、故障排查

### API 文档模板
用于记录接口的详细信息，包含请求参数、返回结果、业务逻辑等。

**使用场景：** 新接口开发、接口变更、接口文档整理
