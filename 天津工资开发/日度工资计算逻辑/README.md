# 日度工资计算逻辑

> 作用：收口天津日度工资 Excel 拆解、日度专项底稿、日度待确认项和后续开发边界。

## 目录

1. [`../最近确认点.md`](../最近确认点.md)：先看当前已定口径收口。
2. [`制管班日度工资Excel拆解.md`](./制管班日度工资Excel拆解.md)：制管班日度 Excel 样例拆解，记录字段、公式、数据源和开发口径。
3. [`日度工资计算模拟案例-当前代码逻辑.md`](./日度工资计算模拟案例-当前代码逻辑.md)：日度工资开发底稿，整理任务节点、场景矩阵和前后端落点。
4. [`日度工资开发待确认-TIME本班组计件.md`](./日度工资开发待确认-TIME本班组计件.md)：日度工资专项的待确认草稿。
5. [`日度工资矩阵与班组分析开发任务.md`](./日度工资矩阵与班组分析开发任务.md)：按财务和班长双视角整理的日度工资矩阵、班组分析、接口、导出和待确认开发方案。

## 当前范围

日度目录只讨论日工资统计、分解、实际发放、支援明细这类日度解释层和追溯层。

月度工资报表里的人员行、月度导出、班组汇总行和月度对账字段，统一回到 [`../月度工资计算逻辑/`](../月度工资计算逻辑/) 讨论，不在日度专项底稿里作为当前月度已完成能力记录。
