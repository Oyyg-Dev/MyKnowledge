# Claude 工作流程规则

## 📋 开发完成后的工作流程

### 1. 功能开发完成检查清单

当完成一个功能开发后，需要确认以下事项：

- [ ] 代码已提交到 Git
- [ ] 功能已测试通过
- [ ] 相关文档已更新
- [ ] 代码已 Review（如需要）

### 2. 工作工时记录提醒

**⚠️ 重要提醒：每次开发完成后，需要记录工作工时！**

#### 记录方式

在开发任务完成后，Claude 会在对话底部提示：

```
✅ 功能开发完成

📊 工作工时记录
是否需要将本次工作记录到工时表中？
- 项目：[项目名称]
- 功能：[功能描述]
- 预估工时：[X小时]
- 日期：[YYYY-MM-DD]

请确认是否记录到：E:\W3-已完成\信息中心AI多维表.csv
```

#### 工时记录格式

记录到 CSV 文件时，使用以下格式：

```csv
年度,月份,日期,工程师,公司事业部,项目,项目功能,总工时/小时,研发工时备注详情,录入人员,录入时间,父记录
2026,5月,2026-05-XX,欧阳彝廣,XX事业部,XX项目,功能描述,X.00,"详细技术实现说明（后端：XH+前端：XH）【日期：2026-05-XX】",欧阳彝廣,2026-05-XX,
```

#### 工时估算原则

1. **简单功能**（UI调整、字段增加）：0.5-1小时
2. **中等功能**（业务逻辑、接口开发）：1-2小时
3. **复杂功能**（重构、多端同步、复杂业务）：2-4小时
4. **Bug修复**：0.5-1小时
5. **系统集成**：1-2小时

#### 工时记录详细说明要求

**研发工时备注详情**应包含：
- 具体技术实现细节
- 使用的技术栈（MinIO、SQL优化、composable等）
- 涉及的模块和文件
- 时间分配（后端：XH+前端：XH+小程序：XH）
- 日期标记【日期：YYYY-MM-DD】

**示例**：
```
1.OA流程状态映射机制重构：修正docStatus映射(30=通过/11=驳回/00=废弃)+ENDED分支状态区分逻辑 
2.MS与小程序数据同步机制优化：统一数据模型+同步接口改造+缓存一致性保障
（后端：4H+小程序：2H）【日期：2026-05-08】
```

### 3. 自动化提醒机制

Claude 在以下情况会自动提醒记录工时：

1. ✅ Git 提交完成后
2. ✅ 功能测试通过后
3. ✅ 用户明确表示"完成"、"搞定"、"OK"等词汇时
4. ✅ 一个完整的开发任务结束时

### 4. 工时记录文件位置

- **主文件**：`E:\W3-已完成\信息中心AI多维表.csv`
- **Excel版本**：`E:\W3-已完成\信息中心AI多维表.xlsx`
- **备份目录**：`E:\W3-已完成\`

### 5. 注意事项

⚠️ **重要**：
- 每天工时不超过 8 小时
- 同一天同一项目的功能可以合并记录
- 工时估算要合理，符合实际开发时间
- 详细描述要体现技术深度和专业性

```

    ↓

    ↓

```
