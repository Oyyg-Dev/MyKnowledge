# T100 多角流程抛转失败问题排查

## 📋 问题概述

**问题描述：** 多角流程抛转失败，接口返回错误信息

**错误代码：**
- 接口返回：`aic-00177` - [T100_message] 多角流程抛转失败
- 真实错误：`aic-00146` - 供应商的采购取价方式为空，请到 apmm202 维护

**影响范围：** 多角流程无法正常抛转，影响业务流程

**发现时间：** 2026-05-06

**解决时间：** 2026-05-06

**状态：** ✅ 已解决

---

## 🔍 问题排查过程

### 第一步：查询失败单据

**查询 xmda_t 表（多角流程单据表）：**

```sql
SELECT xmdaent, xmdasite, xmdadocno, xmdaud002, xmda004, xmda006, xmda050, xmdastus, xmda031 
FROM DSDATA.xmda_t 
WHERE xmdadocno = 'XB-G019-260506000064' 
   OR xmdaud002 = 'XHBJ26050600047' 
ORDER BY xmdadocno
```

**查询结果：**
- 最新失败单号：`XB-G019-260506000064`
- 失败时间：2026-05-06 16:31:30
- 关联单号：`XHBJ26050600047`
- 共查询到 5 条相关记录

---

### 第二步：定位问题根因

**查询 pmab_t 表（供应商据点资料表）：**

```sql
SELECT pmabsite, pmab054, pmab104, pmabstus, pmabmodid, pmabmoddt 
FROM DSDATA.pmab_t 
WHERE pmab001 = 'K010016' 
  AND pmabsite = 'FLD-XB'
```

**查询结果：**
```
PMABSITE   PMAB054   PMAB104   PMABSTUS   PMABMODID   PMABMODDT
--------   -------   -------   --------   ---------   -------------------
FLD-XB     NULL      FLD01     Y          C645        2026-05-05 11:23:34
```

**问题根因：**
- 供应商：`K010016`
- 据点：`FLD-XB`
- **采购取价方式字段（pmab054）为 NULL** ← 问题所在

---

## 💡 解决方案

### 数据补充

**操作步骤：**
1. 登录 T100 系统
2. 进入作业：`apmm202`（供应商据点资料维护）
3. 查询条件：
   - 供应商：`K010016`
   - 据点：`FLD-XB`
4. 补充字（pmab054）
5. 保存并确认

**注意事项：**
- 环境：TOPPRD（正式环境）
- 企业：888
- 确保补充的是正式环境的数据，不是测试环境

## 📊 问题分析

### 为什么会出现这个问题？

1. **数据不完整：** 供应商据点资料维护时，采购取价方式字段未填写
2. **系统校验：** 多角流程抛转时，系统会校验供应商的采购取价方式
3. **错误提示不明确：** 接口返回 aic-00177，需要查询 icer_t 表才能看到真实错误 aic-00146

---

### 如何避免类似问题？

1. **数据维护规范：** 在 apmm202 作业中维护供应商据点资料时，确保必填字段完整
2. **数据校验：** 在供应商据点资料保存时，增加必填字段校验
3. **错误提示优化：** 优化接口错误提示，直接返回真实错误信息

## 📝 经验总结

### 排查技巧

1. **接口错误不一定是真实错误：** 需要查询 icer_t 表获取详细错误信息
2. **数据库直接查询：** 通过 SQL 直接查询相关表，快速定位问题
3. **字段为 NULL 的排查：** 重点检查必填字段是否为空

---

### 沟通技巧

当用户反馈"已经补全数据"但问题仍然存在时，需要：
1. 再次查询数据库，确认数据是否真的已更新
2. 明确告知用户查询结果（字段仍为 NULL）
3. 提供详细的核对口径（环境、企业、供应商、据点、字段）
