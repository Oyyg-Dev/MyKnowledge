# T100 终端删除键和方向键异常排查

## 🔴 问题现象

使用 **xterminal** SSH 登录鼎捷 T100 后出现以下问题：

- **Backspace / Delete** 删除异常（可能出现乱码或 `~`）
- **方向键**移动异常（跳格、最后卡住）

**特点**：同一客户端连接其他服务器正常，优先怀疑 **T100 服务器侧终端/行编辑配置**问题。

---

## 🔍 环境信息

### 系统环境
- **Shell**: `-ksh` (ksh93u+ 2012-08-01)
- **TERM**: `xterm`
- **服务器**: T100 ERP 服务器 (192.168.1.107)
- **SSH 账号**: t100_ssh_ro

### 按键发送情况
使用 `cat -v` 查看按键发送：
- **Delete**: `^[[3~`
- **左键**: `^[[D`
- **右键**: `^[[C`

### terminfo 配置
使用 `infocmp -1 xterm` 查看关键字段：
- `kcub1=\EOD`（左键）
- `kcuf1=\EOC`（右键）

**问题**：实际发送的是 `^[[D/^[[C`，与 terminfo 定义的 `\EOD/\EOC` 不一致。

---

## 🛠️ 排查过程

### 1. 确认问题范围
- ✅ 其他服务器连接正常
- ✅ 其他客户端连接 T100 也有问题
- ❌ 问题出在 T100 服务器侧配置

### 2. 检查终端配置
```bash
# 查看当前 TERM 设置
echo $TERM
# 输出: xterm

# 查看 terminfo 配置
infocmp -1 xterm | grep -E "kcub1|kcuf1|kdch1"
# kcub1=\EOD
# kcuf1=\EOC
# kdch1=\E[3~
```

### 3. 检查 Shell 配置
```bash
# 查看当前 Shell
echo $SHELL
# 输出: -ksh

# 检查 .profile 或 .kshrc
cat ~/.profile
cat ~/.kshrc
```

### 4. 分析根本原因
- 客户端发送的按键码与服务器 terminfo 定义不匹配
- ksh 的行编辑模式可能影响按键处理
- 需要调整服务器侧的终端配置

---

## ✅ 解决方案

### 方案 1: 调整 TERM 环境变量（推荐）
```bash
# 在 ~/.profile 中添加
export TERM=xterm-256color

# 或者临时设置
export TERM=xterm-256color
```

### 方案 2: 使用 stty 配置
```bash
# 设置删除键
stty erase ^?

# 或者
stty erase ^H
```

### 方案 3: 配置 ksh 行编辑模式
```bash
# 在 ~/.profile 中添加
set -o emacs  # 使用 emacs 模式
# 或
set -o vi     # 使用 vi 模式
```

### 方案 4: 客户端配置（如果服务器无法修改）
在 SSH 客户端（如 PuTTY、Xshell）中：
- 设置终端类型为 `xterm-256color`
- 调整键盘映射配置

---

## 🎯 最终采用方案

在 T100 服务器的 `~/.profile` 中添加：
```bash
# 设置终端类型
export TERM=xterm-256color

# 设置删除键
stty erase ^?

# 启用 emacs 行编辑模式
set -o emacs
```

重新登录后问题解决。

---

## 💡 经验总结

### 根本原因
- 终端类型不匹配导致按键码解析错误
- ksh 默认行编辑模式可能不支持某些按键

### 排查思路
1. 确认问题范围（客户端 vs 服务器）
2. 检查终端类型和 terminfo 配置
3. 检查 Shell 配置和行编辑模式
4. 逐个尝试解决方案

### 预防措施
- 统一服务器终端配置
- 在 `.profile` 中设置标准的终端环境
- 文档化服务器配置规范

## 🔗 相关知识

### terminfo 常用命令
```bash
# 查看当前终端信息
infocmp

# 查看特定终端类型
infocmp xterm-256color

# 查看按键定义
```

### stty 常用命令
```bash
# 查看当前设置
stty -a

# 设置删除键
stty erase ^?

# 恢复默认设置
stty sane
```

### ksh 行编辑模式
```bash
# emacs 模式（推荐）
set -o emacs

# vi 模式
set -o vi

# 查看当前模式
set -o | grep -E "emacs|vi"
```
