# 鼎捷 T100：SSH 终端 Delete/Backspace/方向键异常排查（简化版，ksh）
## 1) 现象
使用 **xterminal** SSH 登录鼎捷 T100 后：

+ Backspace / Delete 删除异常（可能出现乱码或 `~`）
+ 方向键移动异常（跳格、最后卡住）

同一客户端连接其他服务器正常，优先怀疑 **T100 服务器侧终端/行编辑配置**问题。

---

## 2) 本次环境确认
+ Shell：`-ksh`（ksh93u+ 2012-08-01）
+ `TERM`：`xterm`
+ `cat -v` 看到按键发送：
    - Delete：`^[[3~`
    - 左键：`^[[D`
    - 右键：`^[[C`
+ `infocmp -1 xterm` 关键字段：
    - `kcub1=\EOD`、`kcuf1=\EOC`（与实际 `^[[D/^[[C` 不一致）
    - `kdch1=\E[3~`

---

## 3) 快速恢复（只影响当前 SSH 会话）
依次执行并测试按键：

```shell
stty sane
stty erase '^?'
stty -ixon
set -o emacs
```

说明：

+ `stty erase '^?'`：修复 Backspace（向左删）常见不匹配
+ `stty -ixon`：避免误触 `Ctrl+S` 导致“像卡住”
+ `set -o emacs`：使用 ksh 的 emacs 行编辑模式

---

## 4) 临时可用但有缺陷的方案（已验证）
> 现象：`export TERM=vt100` 或 `export TERM=vt220` 后，方向键/移动明显正常，但 **Delete 键会输入 **`~`**，无法删除**。
>

原因（简述）：vt100/vt220 的键能力定义与 Delete 行为不完全匹配，导致 `^[[3~` 没被当作“删除”，末尾 `~` 会被当作普通字符插入。

临时使用示例：

```shell
export TERM=vt100   # 或 vt220
stty erase '^?'
stty -ixon
set -o emacs
```

---

## 5) Delete 无法用时的替代键（ksh/emacs）
+ 左移：`Ctrl+B`
+ 右移：`Ctrl+F`
+ 向右删除（删除光标下字符）：`Ctrl+D`
+ 向左删除：Backspace（需先 `stty erase '^?'`）

---

## 6) 建议留存的定位命令（复现/交接给管理员用）
```shell
echo $TERM
echo $0
print -r -- ${KSH_VERSION:-"no KSH_VERSION"}
stty -a | egrep 'erase|ixon'
cat -v
infocmp -1 $TERM | egrep 'kbs=|kcub1=|kcuf1=|kdch1='
```

---

## 7) 结论（本次命中点）
T100 上的 `TERM=xterm` 条目里，左右键定义为 `ESC O D/C`（`\EOD/\EOC`），而 xterminal 实际发送 `ESC [ D/C`（`^[[D/^[[C`），导致 ksh 行编辑对方向键/Delete 解释异常，从而出现跳格/卡住/删除异常等现象。
