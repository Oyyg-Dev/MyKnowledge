# 生产计件 - 技术债务

## 🔴 严重技术债务

### 1. 工资计算逻辑错误
- **问题描述**: 整个计算逻辑都错，目前是根据工种的班组进行计算的
- **影响范围**: 核心业务逻辑，影响工资准确性
- **状态**: 🔴 待重构
- **根本原因**: 
  - 计算逻辑设计不合理
  - 未考虑支援人员场景
  - 班组归属判断错误
- **重构方案**:
  1. 重新梳理工资计算规则
  2. 支援人员的工资以人的部门班组为准
  3. 考虑支援工序切换的场景
  4. 增加单元测试覆盖各种场景
- **相关表**: `bggauc_t` 需要增加字段：是否为拆分项目
- **优先级**: 高
- **预计工作量**: 3-5 天

## 💡 重构建议

### 工资计算重构方案

#### 当前问题
```java
// 错误的逻辑：根据工种的班组计算
if (工种.班组 == "A班") {
    计算工资();
}
```

#### 建议方案
```java
// 正确的逻辑：根据人员的部门班组计算
if (人员.部门班组 == "A班") {
    if (是否支援) {
        // 支援人员特殊处理
        按支援规则计算();
    } else {
        // 正常计算
        按正常规则计算();
    }
}
```

#### 需要考虑的场景
1. **正常计件**: 人员在自己班组工作
2. **支援计件**: 人员支援其他班组
3. **拆分项目**: 多人协作的项目
4. **特殊班组**: 总装班（压装/包装）
